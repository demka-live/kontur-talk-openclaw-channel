# kontur-talk-openclaw-channel

OpenClaw channel plugin for **Kontur Talk (Толк.Чаты)** — корпоративный мессенджер Контура.

Плагин позволяет подключить OpenClaw к чат-боту в Толк.Чаты для взаимодействия с AI-ассистентом через привычный мессенджер.

## Возможности

- Получение сообщений из комнат и личных чатов через Long Polling
- Отправка текстовых ответов с поддержкой Markdown-форматирования
- Загрузка и отправка изображений
- Поддержка тредов (обсуждений)
- Управление доступом через DM-политику (allowlist / open)
- Настройка обязательного @-упоминания в групповых чатах

## Предварительные требования

1. **Node.js** >= 22
2. **OpenClaw** установлен и настроен
3. **Чат-бот в Толк.Чаты** — создайте бота через интерфейс Толк.Чаты на экране настроек пространства и получите JWT-токен

## Получение JWT-токена бота

1. Откройте Толк.Чаты
2. Перейдите на экран вашего **пространства**
3. Нажмите кнопку **создания чат-бота**
4. Скопируйте выданный **JWT-токен**

Токен содержит:
```json
{
  "sub": "{ID бота}",
  "owner": "{ID администратора бота}",
  "iat": 1757061777
}
```

## Установка в OpenClaw

### Вариант 1: Установка из GitHub Release (рекомендуемый)

Скачайте готовый архив из [Releases](../../releases/latest) и установите:

```bash
# Скачайте и распакуйте последний релиз
curl -L https://github.com/demka-live/kontur-talk-openclaw-channel/releases/latest/download/kontur-talk.tar.gz -o kontur-talk.tar.gz
tar -xzf kontur-talk.tar.gz

# Установите плагин
openclaw plugins install ./kontur-talk/
```

### Вариант 2: Сборка из исходников

```bash
# Клонируйте репозиторий
git clone https://github.com/demka-live/kontur-talk-openclaw-channel.git
cd kontur-talk-openclaw-channel

# Установите зависимости и проверьте типы
npm install
npm run check

# Установите плагин из текущей директории
openclaw plugins install ./
```

## Конфигурация

После установки плагина настройте его в файле конфигурации OpenClaw (`openclaw.json`):

```json5
{
  channels: {
    "kontur-talk": {
      // (обязательно) JWT-токен бота из Толк.Чаты
      "token": "eyJhbGciOi...",

      // (опционально) Базовый URL API, по умолчанию https://chat.ktalk.ru
      "baseUrl": "https://chat.ktalk.ru",

      // (опционально) Таймаут long polling в секундах (1-60), по умолчанию 30
      "pollingTimeout": 30,

      // (опционально) Политика доступа к DM: "allowlist" или "open"
      "dmSecurity": "allowlist",

      // (опционально) Список разрешённых пользователей для DM
      "allowFrom": [
        "@alice:ktalk.ru",
        "@bob:ktalk.ru"
      ],

      // (опционально) Требовать @-упоминание бота в групповых чатах (по умолчанию true)
      "requireMention": true
    }
  }
}
```

### Переменные окружения

Вместо указания токена напрямую в конфигурации, можно использовать переменную окружения:

```bash
export KONTUR_TALK_BOT_TOKEN="eyJhbGciOi..."
```

### Интерактивная настройка

```bash
openclaw onboard
```

Мастер настройки предложит ввести токен бота и базовый URL.

## Использование

После настройки плагин запускается автоматически вместе с OpenClaw:

```bash
openclaw start
```

Бот начнёт получать сообщения из Толк.Чаты через Long Polling и отвечать через OpenClaw.

### Как это работает

1. Плагин запускает фоновый сервис, который делает Long Polling запросы к API Толк.Чаты
2. Новые сообщения проходят через фильтр упоминаний (в групповых чатах)
3. Сообщения передаются в конвейер OpenClaw для обработки AI-моделью
4. Ответ AI отправляется обратно в чат через Bot API

### Особенности поведения

- **Личные чаты**: бот отвечает на все сообщения (с учётом DM-политики)
- **Групповые чаты**: бот отвечает только при @-упоминании (настраивается через `requireMention`)
- **Треды**: бот поддерживает ответы в тредах, сохраняя контекст обсуждения
- **Форматирование**: ответы отправляются в Markdown-формате

## API Толк.Чаты

Плагин использует следующие эндпоинты Bot API:

| Эндпоинт | Метод | Описание |
|-----------|-------|----------|
| `/get_updates` | GET | Long Polling для получения новых сообщений |
| `/send_message` | POST | Отправка текстового сообщения |
| `/get_messages_in_thread` | POST | Получение сообщений из треда |
| `/upload_image` | POST | Загрузка изображения на сервер |
| `/download_media` | POST | Скачивание медиафайла по MXC URL |

Базовый URL API: `https://chat.ktalk.ru/_matrix/client/strangler/api/v1/bot/{jwt_token}/...`

## Устранение неполадок

### Бот не получает сообщения

1. Убедитесь, что JWT-токен действителен
2. Проверьте, что бот добавлен в пространство
3. Убедитесь, что бот приглашён в комнату или создан личный чат
4. Проверьте сетевую доступность `https://chat.ktalk.ru`

### Бот не отвечает в групповом чате

1. Убедитесь, что упоминаете бота через @
2. Или установите `"requireMention": false` в конфигурации

### Ошибка 401 Unauthorized

JWT-токен недействителен или истёк. Создайте нового бота и обновите токен.

### Ошибка 403 Forbidden

Бот не имеет прав для отправки сообщений в комнату. Обратитесь к администратору пространства.
